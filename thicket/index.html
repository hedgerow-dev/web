Runtime protection Closed beta

Your AI application can be tricked into running code. Thicket stops it.

Prompt injection attacks don’t announce themselves — they arrive disguised as user input, tool results, or RAG content and route LLM output toward subprocesses, network calls, or file writes. Thicket sits between your application and everything dangerous it could do, and blocks those operations at the execution boundary before they complete.

No external API Runs entirely in-process. No outbound calls, no latency overhead, no data leaving your environment.

What Thicket blocks

The attacks your existing tools can’t see.

Cloud prompt scanners inspect text and return a score. Thicket intercepts execution — the actual syscall, the actual network connection — so there’s no prompt obfuscation that bypasses it.

Prompt injection

Scans every prompt, tool result, and model output before it can trigger downstream execution. In-process. No content leaves your environment.

Code execution via LLM output

Intercepts exec, eval, and subprocess calls at the Python level before they run, regardless of how the call was reached.

SSRF & network abuse

Blocks outbound connections to private IP ranges and cloud metadata endpoints — enforced at the network call, not the prompt.

File system abuse

Sandboxes write access to sensitive paths — credentials, SSH keys, cloud config — so a compromised agent can’t exfiltrate them.

Model supply chain

Scans model checkpoint files for embedded payloads before they are loaded — stopping the attack before the Python runtime sees it.

OS kernel backstop

A second, independent enforcement layer at the kernel level. Cannot be reversed from application code. Catches bypasses that evade the Python layer.

Deployment

One library. Drop-in for the frameworks you already use.

Thicket installs as a standard Python package and auto-patches LangChain, LlamaIndex, FastAPI, Flask, and Django. Run in audit_only mode to validate false positive rates before switching to enforcement.
