How it works

Defence in depth, from prompt to kernel.

Each enforcement layer operates independently at a different point in the execution path. An attacker who evades one layer immediately faces the next, with different bypass requirements.

1

Prompt & content scanning

Every prompt, tool call result, and model output is scanned against deterministic rule sets before it can trigger downstream execution. No content leaves your process. Catches injection attempts before they reach an operation that can cause harm.

2

Runtime interception

Hooks into the Python runtime to intercept dangerous operations (code execution, network calls, file writes, subprocess spawning) at the moment they are attempted. No prompt obfuscation bypasses a hook on the operation itself. Works regardless of how the call was reached.

3

OS kernel backstop

A second, independent enforcement layer operating at the kernel level on Linux. Completely separate from the Python runtime. An exploit that disables the application-level layer still hits this one. Cannot be reversed from application code.
