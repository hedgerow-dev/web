About

AI infrastructure introduces a class of attack surfaces that traditional security tools were not designed to model: unsafe model loading, prompt injection, trust boundary violations, multi-hop exploit chains across LLM frameworks.

Our position: security controls for AI applications should be deterministic, deployable without external API calls, and fully auditable — regardless of the environment they run in.

Products

Thicket Runtime security

Four independent enforcement layers active inside your Python application. Blocks attacks at the execution boundary — network calls, file writes, process spawning — before they complete. An attacker who bypasses one layer immediately hits the next.

  • Blocks SSRF, deserialization RCE, prompt injection, and agent exploits
  • No AI Inside™ — every block traces to a rule, a patch, or a kernel control
  • No external API. No data leaves the process.
  • OS kernel backstop on Linux ≥ 5.10; eBPF DaemonSet for Kubernetes (early access)

Rowan Source code analysis

Cross-file taint propagation and AI-specific sink models across four parallel engines. Finds unsafe model loading paths, prompt injection chains, and multi-hop exploit patterns that pattern-based scanners miss. Confirmed critical RCEs in production AI frameworks.

  • Python-first; Go, Java, JavaScript, C# in progress

The approach

01

Deterministic enforcement, not probability scores.

Every blocking decision traces to a rule, a patch, or a kernel control. Not a model confidence score. That distinction matters to auditors, compliance teams, and any team that needs to explain a block — or a miss — to a regulator.

02

No AI in the security decision path.

Enforcement does not depend on another language model. There is no external API call, no cloud service, no training corpus that includes your production data. The logic that makes blocking decisions runs locally, deterministically, with no network dependency.

03

In-process, in your environment.

Thicket installs inside your application process. Rowan runs on your infrastructure. Neither product requires a network connection or a third-party service. Air-gapped and regulated environments are the design target, not a deployment afterthought.
